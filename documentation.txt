1. System Overview
1.1 Project Description
A dual-platform web application consisting of:

Visitor Website: Public-facing platform showcasing organizational content

Admin Dashboard: Secure control panel for content management and system administration

1.2 Core Philosophy
Separation of Concerns: Clear distinction between presentation and administration

Real-time Synchronization: Changes in admin dashboard immediately reflect on visitor site

Responsive Design: Optimal experience across all device sizes

Professional UX/UI: Clean, intuitive interfaces with consistent theming

2.2 Technology Stack
Framework: Next.js 14 (App Router)

Styling: Tailwind CSS with CSS Modules

Database: Supabase (PostgreSQL)

Authentication: Supabase Auth with Row Level Security

Deployment: Vercel (Primary), Render (Backup)

Payment Integration: Custom gateway with MTN/Airtel Mobile Money

Analytics: Custom implementation with charting libraries

5. Database Schema Design
5.1 Core Tables
sql
-- Users & Authentication
admins (
  id UUID PRIMARY KEY,
  email VARCHAR UNIQUE NOT NULL,
  password_hash VARCHAR NOT NULL,
  full_name VARCHAR NOT NULL,
  profile_image_url VARCHAR,
  phone_number VARCHAR,
  role VARCHAR DEFAULT 'admin',
  is_active BOOLEAN DEFAULT true,
  last_login TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
)

-- Site Configuration
site_config (
  id UUID PRIMARY KEY,
  config_key VARCHAR UNIQUE NOT NULL,
  config_value JSONB NOT NULL,
  description TEXT,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
)

-- Theme Settings
theme_settings (
  id UUID PRIMARY KEY,
  name VARCHAR NOT NULL,
  primary_color VARCHAR DEFAULT '#1E40AF',
  background_color VARCHAR DEFAULT '#FFFFFF',
  text_color VARCHAR DEFAULT '#000000',
  font_family VARCHAR DEFAULT 'system-ui',
  is_active BOOLEAN DEFAULT true,
  created_by UUID REFERENCES admins(id),
  created_at TIMESTAMP DEFAULT NOW()
)
5.2 Content Tables
sql
-- Hero Slides
hero_slides (
  id UUID PRIMARY KEY,
  image_url VARCHAR NOT NULL,
  description TEXT,
  display_order INTEGER NOT NULL,
  transition_direction VARCHAR DEFAULT 'left-to-right',
  duration_seconds INTEGER DEFAULT 5,
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
)

-- Vision, Mission, Objectives
content_sections (
  id UUID PRIMARY KEY,
  section_type VARCHAR NOT NULL, -- 'vision', 'mission', 'objective'
  title VARCHAR,
  content TEXT NOT NULL,
  image_url VARCHAR,
  display_order INTEGER,
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
)

-- Programs
programs (
  id UUID PRIMARY KEY,
  title VARCHAR NOT NULL,
  short_description TEXT,
  long_description TEXT,
  featured_image_url VARCHAR,
  start_date DATE,
  end_date DATE,
  location VARCHAR,
  target_audience VARCHAR,
  status VARCHAR DEFAULT 'draft',
  created_by UUID REFERENCES admins(id),
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
)

-- Gallery
gallery_items (
  id UUID PRIMARY KEY,
  image_url VARCHAR NOT NULL,
  title VARCHAR,
  description TEXT,
  category VARCHAR,
  tags TEXT[],
  display_order INTEGER,
  is_featured BOOLEAN DEFAULT false,
  created_at TIMESTAMP DEFAULT NOW()
)

-- News
news_articles (
  id UUID PRIMARY KEY,
  title VARCHAR NOT NULL,
  excerpt TEXT,
  content TEXT NOT NULL,
  featured_image_url VARCHAR,
  author_id UUID REFERENCES admins(id),
  status VARCHAR DEFAULT 'draft',
  published_at TIMESTAMP,
  views_count INTEGER DEFAULT 0,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
)
5.3 User Interaction Tables
sql
-- Contact Form Submissions
contact_submissions (
  id UUID PRIMARY KEY,
  full_name VARCHAR NOT NULL,
  email VARCHAR NOT NULL,
  phone_number VARCHAR NOT NULL,
  gender VARCHAR,
  residence VARCHAR,
  subject VARCHAR,
  message TEXT NOT NULL,
  status VARCHAR DEFAULT 'new',
  admin_notes TEXT,
  resolved_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW()
)

-- Donations
donations (
  id UUID PRIMARY KEY,
  donor_name VARCHAR NOT NULL,
  donor_email VARCHAR,
  donor_phone VARCHAR,
  amount DECIMAL NOT NULL,
  currency VARCHAR DEFAULT 'USD',
  payment_method VARCHAR NOT NULL, -- 'mobile_money', 'card', 'direct_mobile'
  payment_status VARCHAR DEFAULT 'pending',
  transaction_id VARCHAR UNIQUE,
  receipt_url VARCHAR,
  created_at TIMESTAMP DEFAULT NOW()
)

-- Website Analytics
page_views (
  id UUID PRIMARY KEY,
  page_path VARCHAR NOT NULL,
  visitor_id VARCHAR,
  session_id VARCHAR,
  device_type VARCHAR,
  referrer VARCHAR,
  country VARCHAR,
  viewed_at TIMESTAMP DEFAULT NOW()
)

Conclusion
This documentation outlines a comprehensive, professional web application with clear separation between visitor-facing content and administrative control. The system is designed with clean architecture principles, following best practices for security, scalability, and maintainability.

The implementation will use modern web technologies with a focus on:

User Experience: Intuitive interfaces with clear feedback

Performance: Fast loading times and smooth interactions

Maintainability: Clean code structure and comprehensive documentation

Scalability: Architecture that supports growth and additional features

Security: Robust protection for both user data and admin access

The system is ready for implementation with the provided specifications, and can be developed incrementally following agile methodologies with regular stakeholder feedback cycles.
